# 01. 자료구조와 알고리즘

## 1. 자료구조란?

자료구조(Data Structure)는 데이터를 효율적으로 저장하고, 관리하고, 사용할 수 있도록 정리하는 방식입니다.  
단순히 데이터를 담는 그릇이 아니라, **검색, 삽입, 삭제, 정렬, 순회** 같은 작업을 얼마나 효율적으로 수행할 수 있는지를 결정합니다.

예를 들어 같은 데이터라도 다음처럼 목적에 따라 적합한 구조가 달라집니다.

| 상황 | 적합한 자료구조 |
|---|---|
| 순서대로 데이터를 저장하고 싶다 | 리스트 |
| 최근 작업을 먼저 되돌리고 싶다 | 스택 |
| 먼저 온 요청을 먼저 처리하고 싶다 | 큐 |
| 계층 구조를 표현하고 싶다 | 트리 |
| 복잡한 연결 관계를 표현하고 싶다 | 그래프 |
| 빠른 검색이 필요하다 | 해시 테이블 |

## 2. 알고리즘이란?

알고리즘(Algorithm)은 문제를 해결하기 위한 절차나 방법입니다.  
자료구조가 데이터를 담는 방식이라면, 알고리즘은 그 데이터를 이용해 문제를 해결하는 과정입니다.

```text
프로그램 = 자료구조 + 알고리즘
```

좋은 프로그램을 만들기 위해서는 적절한 자료구조 선택과 효율적인 알고리즘 설계가 함께 필요합니다.

## 3. 알고리즘의 조건

알고리즘은 보통 다음 조건을 만족해야 합니다.

| 조건 | 의미 |
|---|---|
| 입력 | 0개 이상의 입력이 존재할 수 있음 |
| 출력 | 1개 이상의 결과가 나와야 함 |
| 명백성 | 각 단계의 의미가 모호하지 않아야 함 |
| 유한성 | 유한한 단계 후 종료되어야 함 |
| 유효성 | 각 명령은 실제로 실행 가능한 연산이어야 함 |

## 4. 선형 자료구조와 비선형 자료구조

자료구조는 크게 선형 구조와 비선형 구조로 나눌 수 있습니다.

## 선형 자료구조

데이터가 일렬로 나열되는 구조입니다.

| 구조 | 특징 |
|---|---|
| 리스트 | 순서가 있는 데이터 저장 |
| 스택 | 마지막에 들어온 데이터가 먼저 나감 |
| 큐 | 먼저 들어온 데이터가 먼저 나감 |
| 덱 | 양쪽 끝에서 삽입과 삭제 가능 |

## 비선형 자료구조

데이터가 한 줄로 단순하게 나열되지 않는 구조입니다.

| 구조 | 특징 |
|---|---|
| 트리 | 계층 구조 표현 |
| 그래프 | 객체 사이의 복잡한 관계 표현 |

## 5. ADT란?

ADT(Abstract Data Type, 추상 자료형)는 자료구조를 **어떻게 구현할지**보다 **무엇을 할 수 있는지**에 초점을 둔 개념입니다.

예를 들어 스택 ADT는 내부 구현이 리스트인지 연결 리스트인지는 중요하지 않고, 다음 연산을 제공한다는 점이 중요합니다.

```text
push(item)   # 삽입
pop()        # 삭제
peek()       # top 확인
is_empty()   # 비어 있는지 확인
```

즉, ADT는 사용자에게 필요한 기능만 공개하고 내부 구현은 숨기는 방식입니다.

## 6. 핵심 정리

- 자료구조는 데이터를 저장하고 관리하는 방식이다.
- 알고리즘은 문제를 해결하는 절차이다.
- 프로그램은 자료구조와 알고리즘의 조합으로 볼 수 있다.
- ADT는 자료구조의 기능을 추상적으로 정의하는 개념이다.
- 선형 자료구조는 리스트, 스택, 큐, 덱이 대표적이다.
- 비선형 자료구조는 트리와 그래프가 대표적이다.
