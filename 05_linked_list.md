# 05. 연결 리스트

## 1. 연결된 구조란?

연결된 구조는 데이터들이 메모리상에 연속적으로 저장되지 않아도, 링크를 이용해 서로 연결된 구조입니다.

배열은 연속된 공간에 데이터를 저장하지만, 연결 리스트는 각각의 노드가 다음 노드를 가리키는 방식으로 구성됩니다.

## 2. 노드(Node)

노드는 일반적으로 데이터 필드와 링크 필드로 구성됩니다.

```python
class Node:
    def __init__(self, data, link=None):
        self.data = data
        self.link = link
```

| 구성 요소 | 설명 |
|---|---|
| data | 실제 저장할 값 |
| link | 다음 노드를 가리키는 참조 |

## 3. 단순 연결 리스트

단순 연결 리스트는 각 노드가 다음 노드를 가리키는 링크 하나를 갖습니다.

```text
head -> [10 | next] -> [20 | next] -> [30 | None]
```

장점은 삽입과 삭제 시 링크만 조정하면 된다는 점입니다.  
단점은 특정 위치에 바로 접근할 수 없고, 앞에서부터 순차적으로 이동해야 한다는 점입니다.

## 4. 삽입 연산

맨 앞에 노드를 삽입하는 과정은 다음과 같습니다.

```python
new_node.link = head
head = new_node
```

기존 첫 번째 노드를 새 노드가 가리키게 만들고, head를 새 노드로 바꿉니다.

## 5. 삭제 연산

특정 노드를 삭제할 때는 이전 노드가 삭제 대상의 다음 노드를 가리키도록 링크를 바꿉니다.

```python
prev.link = target.link
```

## 6. 원형 연결 리스트

원형 연결 리스트는 마지막 노드가 다시 첫 번째 노드를 가리키는 구조입니다.

```text
[10] -> [20] -> [30]
 ^             |
 |-------------|
```

큐를 연결 구조로 구현할 때 활용할 수 있습니다.

## 7. 이중 연결 리스트

이중 연결 리스트는 각 노드가 이전 노드와 다음 노드를 모두 가리킵니다.

```text
None <- [10] <-> [20] <-> [30] -> None
```

장점은 양방향 이동이 가능하다는 점입니다.  
단점은 링크가 두 개이므로 메모리 사용량이 증가하고 구현이 복잡해진다는 점입니다.

## 8. 연결 리스트와 배열 리스트 비교

| 항목 | 배열 리스트 | 연결 리스트 |
|---|---|---|
| 인덱스 접근 | 빠름 O(1) | 느림 O(n) |
| 맨 앞 삽입 | 느림 O(n) | 빠름 O(1) |
| 중간 삽입/삭제 | 이동 비용 발생 | 위치만 알면 링크 조정 |
| 메모리 | 연속 공간 필요 | 링크 저장 공간 추가 필요 |

## 9. 핵심 정리

- 연결 리스트는 노드와 링크로 구성된다.
- 단순 연결 리스트는 한 방향으로만 이동한다.
- 원형 연결 리스트는 마지막 노드가 첫 노드를 가리킨다.
- 이중 연결 리스트는 양방향 이동이 가능하다.
- 연결 리스트는 삽입/삭제에 유리하지만 임의 접근은 느리다.
